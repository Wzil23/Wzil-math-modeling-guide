步骤 1/4：整合方案（合并前置合同 + 设计理论 + 常见模式）

→ 读取 references/roles/建模手/references/建模设计理论.md，补充选择依据：

  - 复杂度控制（2 模型法则、3 层嵌套、5 参数法则）

→ 读取 references/roles/建模手/references/前置合同.md，将结果组织成结构化合同：
  - 子问题-模型映射表（每个子问题对应哪个模型、承担什么证据角色）

步骤 3/4：执行建模分析
→ 读取 references/roles/建模手/references/工作流程.md
→ 按流程执行：
  - 题目理解：确认问题背景、类型、数据、约束（mmstep1 已读取，此处确认无遗漏）
  - 模型选择：深化公式推导，给出完整的数学表达式、参数含义、取值范围
  - 算法设计：详细的求解步骤、输入输出格式、边界条件处理
  - 术语规范：建立中英文术语对照表、符号定义
→ 题目中如有示意图、图表等图像内容，必须派发子代理并指定 model: "opus" 识别
→ 产出：
  - 题目分析报告.md（按工作流程.md 的文档模板格式）
  - 术语表格.md
→ 推荐模型说明图方案（2-3 种），说明每种图的内容和用途：
  - 模型架构图：展示模型整体结构和各模块关系
  - 算法流程图：展示求解算法的步骤和逻辑
  - 几何关系图：展示空间关系、约束条件（如有几何/空间类问题）
  - 其他有助于理解模型的概念示意图

→ 绘制完成后，通过 Skill 工具调用 nature-figure skill 对图表做 QA 检查（配色、字体、分辨率、坐标轴规范），不达标的修正
→ 完成后报告：各问题的模型和公式概要、生成的模型说明图文件列表
→ ⏸️ 暂停，等用户确认后进入下一步
