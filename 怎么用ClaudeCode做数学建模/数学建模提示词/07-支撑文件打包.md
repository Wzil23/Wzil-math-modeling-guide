进入支撑材料打包阶段。

前置读取：
- 读取论文（paper/ 目录下的 .docx 或 .tex 文件），确认附录中引用了哪些支撑材料
- 读取以下目录的内容：
  - code/（求解代码）
  - results/（结果表格）
  - figures/（图表文件）
  - 题目分析报告.md、术语表格.md（项目根目录）

⚠️ 必须严格按以下顺序逐步执行，不得跳步、合并或自行调整顺序。每完成一步后必须暂停等用户确认，确认后才能进入下一步。

步骤 1/5：收集支撑材料文件
→ 代码文件：code/ 目录下所有源程序（必须是完整可运行的，包含所有依赖说明）
→ 数据资料：自行查阅使用的外部数据（注意：赛题提供的原始数据不包含在内）
→ 中间结果：results/ 目录下的详细数据表格
→ 图表文件：figures/ 目录下较大篇幅的中间结果图表
→ 分析文档：题目分析报告.md、术语表格.md（用于一致性检查）
→ 完成后报告：收集到的文件列表和总大小
→ ⏸️ 暂停，等用户确认后进入下一步

步骤 2/5：检查文件合规性
→ 确认所有文件中没有显示参赛者身份、姓名、学校名称、赛区信息
→ 确认没有包含竞赛承诺书和编号专用页
→ 确认支撑材料与论文内容相符（支撑材料中的代码和数据应与论文附录中描述的一致）
→ 如有不合规内容，清理后重新检查
→ 完成后报告：合规性检查结果（通过/有问题需修复）
→ ⏸️ 暂停，等用户确认后进入下一步

步骤 3/5：检查文件完整性并处理论文附录
→ 核对论文附录中引用的支撑材料是否全部包含
→ 检查源程序是否可运行（依赖库是否齐全、是否有说明文档）
→ 如果有支撑材料：将支撑材料的文件列表（文件名 + 文件说明）写入论文附录
→ 如果没有需要提供的支撑材料：在论文附录中注明"本论文没有支撑材料"，跳过步骤 4 和 5
→ 完成后报告：完整性检查结果、论文附录已更新
→ ⏸️ 暂停，等用户确认后进入下一步

步骤 4/5：压缩打包
→ 将所有支撑材料文件压缩为一个压缩包
→ 格式：ZIP（优先）或 RAR
→ 压缩包大小不超过 20MB
→ 如果超过 20MB，检查是否有不必要的大文件可以精简，告知我后决定处理方式
→ 产出：支撑材料.zip（项目根目录）
→ 完成后报告：压缩包文件名、大小、包含的文件数量
→ ⏸️ 暂停，等用户确认后进入下一步

步骤 5/5：最终确认
→ 确认支撑材料.zip 内容正确
→ 确认论文附录中的文件列表与压缩包内容一致
→ 确认压缩包大小不超过 20MB
→ 完成后报告：打包完成，最终文件列表和大小
→ ⏸️ 暂停，等用户确认
