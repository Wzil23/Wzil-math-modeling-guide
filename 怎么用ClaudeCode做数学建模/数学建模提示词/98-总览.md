# 数学建模提示词 - 总览

## 使用方式

### 方式1：Slash Command（推荐）

在 Claude Code 中输入以下命令，每个阶段完成后等 Claude 产出结果并确认后再进入下一步：

| 命令 | 做什么 |
|------|--------|
| `/mmsetup` | 配置语言、格式、比赛类型（每次新题发一次） |
| `/mmstep1` | 读取题目和数据 |
| `/mmstep2` | brainstorming 方案探索 |
| `/mmstep3` | writing-plans 任务拆分 |
| `/mmstep4` | math-modeling 建模手 |
| `/mmstep5` | math-modeling 编程手 + TDD（含并行选项） |
| `/mmstep6` | math-modeling 论文手（含并行选项） |
| `/mmstep7` | 打包支撑材料 |

详细方案见文件"部署SlashCommand.md"

### 方式2：复制粘贴

从 `C:\Users\......\Desktop\数学建模提示词\` 下的 `.md` 文件中复制代码块内容到对话框。

### 方式3：一次性发送

复制 `99-一次性发送版.md` 中的代码块内容，一次性发给 Claude。

## 设计原则

1. **每个阶段只调用一次分析**：brainstorming 做方案探索，math-modeling 建模手做深度建模，不重复
2. **skill 调用明确**：每个步骤写明具体调用哪个 skill，不写"记得使用相关 skill"
3. **阶段职责不重叠**：分析归分析，计划归计划，执行归执行
4. **TDD 按需使用**：只对核心算法函数做 TDD，脚本代码不做
5. **subagent 有明确触发条件**：独立子问题 >= 2 个时才用
6. **通用化**：所有文件名、语言等通过 `/setup` 或 Claude 主动询问确定，提示词本身不含占位符
