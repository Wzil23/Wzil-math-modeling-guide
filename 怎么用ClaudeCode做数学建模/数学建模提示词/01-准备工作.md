⚠️ 主代理没有多模态能力，不能直接读取或识别任何图片文件（.png/.jpg/.bmp等）。遇到图片必须派发子代理并指定 model: "opus" 处理。

1. 题目文件名（不限格式：PDF/Word/图片等）
2. 格式要求文件名（不限格式：PDF/Word/图片等）
3. 附件数据文件名（Excel/CSV），如果没有附件数据用户会说"没有"

等用户回答后，依次执行：
- 根据文件扩展名自动选择工具读取题目文件：
  - PDF → 先用 pdf skill 提取文字部分。如果 PDF 中包含示意图、图表、流程图、拓扑图、手写公式等图像内容，必须派发子代理并指定 model: "opus" 识别图像部分

  - 图片 (.png/.jpg) → 必须派发子代理并指定 model: "opus" 识别
  - 其他格式 → 告知用户，由用户决定处理方式
- 同样方式读取格式要求文件
- 如果有附件数据，使用 xlsx skill 读取（读取时注意中文编码，确保不出现乱码）

⚠️ 派发 opus 子代理识别图像时，必须在 prompt 中明确要求以下内容（不能只说"读取图片"）：
  1. 先判断图片类型（系统拓扑图/流程图/几何示意图/数据图表/公式约束/参数表/地图空间/图例编码），不同类型提取重点不同

  4. 描述拓扑/连接关系（谁连谁、箭头方向、边的标签）
  5. 提取图例/颜色编码含义
  6. 提取标注的参数值、单位、约束条件
  7. 如有子图(a)(b)(c)，逐个识别

- 题目有几个小问
- 每个小问的核心要求
- 附件数据的结构和规模（如有）
