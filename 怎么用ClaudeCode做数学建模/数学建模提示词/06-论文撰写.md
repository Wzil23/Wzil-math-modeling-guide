步骤 1/14：构建论证框架
→ 读取 references/roles/论文手/references/工作流程.md
→ 按论证优先流程执行：
  - 构建论证链：用一句话论证模板概括全文（针对 [问题]，使用 [方法]，达到 [结果]，由 [证据] 支撑，适用于 [边界]）
  - 核心要素：论点（claim）、证据（evidence）、边界（boundary）、意义（significance）
  - Claim-Evidence 映射：每个核心论点必须映射到具体的图表/数据证据，标注状态（已支撑/待补充）
→ 缺失输入处理规则：
  - 数据不足 → 用 `[待补充：数据来源]` 占位，不编造数据
  - 参考文献不确定 → 用 `[待补充：文献]` 占位，不编造引用
  - 结果未计算 → 用 `[待补充：数值]` 占位，不编造结果
→ 完成后报告：论证框架概要（一句话论证 + Claim-Evidence 映射表）
→ ⏸️ 暂停，等用户确认后进入下一步

步骤 2/14：读取章节模板，规划论文结构
→ 读取 references/roles/论文手/references/章节模板.md
→ 按模板组织各章节结构：
  - 摘要：六要素（背景→缺口→方法→结果→意义→边界），300-1000 字
  - 引言：五要素（领域规模→瓶颈→已有尝试→缺口→本研究）
  - 模型建立：按子问题分节，每节含问题分析、模型选择理由、数学模型、求解方法
  - 结果分析：证据阶梯（体系→验证→主结果→对比→分析→应用），每张图配 100+ 字分析
  - 模型评价：进阶→证据意义→与已有工作对比→局限→未来应用
→ 完成后报告：论文大纲框架
→ ⏸️ 暂停，等用户确认后进入下一步

步骤 3/14：读取写作规范

步骤 10/14：通过 Skill 工具调用 nature-reviewer skill，对论文进行审稿人视角的审查

       · 跨学科读者兴趣：结论是否对非本领域读者有意义
       · 技术可靠性：识别必须在论点确立前解决的技术缺陷
