# Claude Code 数学建模实战指南

> 从零开始，用 Claude Code 完成一份数学建模竞赛论文的完整流程。

---

## 一、前置准备

### 1.1 安装 Claude Code

Claude Code 是 Anthropic 官方的命令行 AI 工具，支持终端、桌面应用、VS Code 插件等。

**方式一：命令行安装（推荐）**

```bash
# 需要先安装 Node.js（v18+）
npm install -g @anthropic-ai/claude-code
```

**方式二：桌面应用**

从 [claude.ai/code](https://claude.ai/code) 下载桌面版（Mac/Windows）。

**方式三：VS Code 插件**

在 VS Code 扩展商店搜索 "Claude Code" 安装。

**首次启动：**

```bash
# 终端中运行
claude
```

首次使用需要登录 Anthropic 账号或配置 API Key。
也可以通过安装CC-switch（`npm install -g cc-switch`），然后切换到国产 API（如 DeepSeek、MiMo 等）。

### 1.2 安装所需 Skill

本流程依赖以下 skill，需要在 Claude Code 中安装：

| Skill | 类型 | 用途 |
|-------|------|------|
| `math-modeling` | 自定义 skill | 建模手、编程手、论文手三阶段流程（含完整 references 目录） |
| `brainstorming` | superpower skill | 题目分析、方案探索 |
| `writing-plans` | superpower skill | 制定实现计划 |
| `subagent-driven-development` | superpower skill | 并行处理独立子问题 |
| `nature-reviewer` | nature skill | 审稿人视角审查论文 |
| `nature-figure` | nature skill | 图表质量检查 |
| `stop-slop` | 自定义 skill | 去 AI 味检查和润色 |

安装方法：将 skill 文件夹放到 `~/.claude/skills/` 目录下。

### 1.3 安装 Python 环境

数学建模通常使用 Python，需要安装以下依赖：

```bash
# 推荐 Python 3.10+
pip install numpy pandas matplotlib scipy openpyxl
```

验证安装：

```bash
python -c "import numpy, pandas, matplotlib, scipy, openpyxl; print('OK')"
```

### 1.4 安装 LaTeX（可选，国赛可用 Word）

如果选择 LaTeX 格式：

```bash
# Windows: 安装 MiKTeX 或 TeX Live
# macOS: 安装 MacTeX
# Linux: sudo apt install texlive-full
```

验证：

```bash
xelatex --version
```

### 1.5 部署 Slash Command

将"ClaudeCode数学建模提示词"文件夹中的 `.md` 文件部署为 Claude Code 的 slash command。

具体步骤参见：`部署SlashCommand.md`

部署完成后，你将拥有以下命令：`/mmsetup`、`/mmstep1` 到 `/mmstep7`

### 1.6 准备工作目录

```bash
mkdir 数学建模项目
cd 数学建模项目
```

把题目 PDF、附件数据（Excel）放到这个目录下。

## 二、工作流程概览

整个流程分为 8 个阶段，每个阶段对应一个斜杠命令：

```
/mmsetup    → 配置阶段（语言、格式、比赛类型）
/mmstep1    → 准备阶段（读取题目和数据）
/mmstep2    → 分析阶段（探索建模思路）
/mmstep3    → 计划阶段（制定实现计划）
/mmstep4    → 建模阶段（建立数学模型）
/mmstep5    → 编程阶段（编写求解代码）
/mmstep6    → 论文阶段（撰写论文 + 自检润色）
/mmstep7    → 打包阶段（打包支撑材料）
```

每个阶段完成后 Claude 会暂停，等你确认后再进入下一阶段。

## 三、详细步骤

### 步骤 1：配置（/mmsetup）

在终端中启动 Claude Code 后，输入：

```
/mmsetup
```

Claude 会问你 4 个问题：

| 问题 | 选项 | 建议 |
|------|------|------|
| 编程语言 | Python / MATLAB / 其他 | 国赛建议 Python |
| 论文格式 | Word / LaTeX | 国赛建议 Word（方便排版），美赛建议 LaTeX |
| 比赛类型 | 国赛 / 美赛 / 其他 | 根据实际选择 |
| TDD 范围 | 不需要 / 核心算法 TDD | 建议"核心算法 TDD" |

配置完成后，Claude 会记住这些设置，后续步骤自动使用。

### 步骤 2：读取题目（/mmstep1）

```
/mmstep1
```

Claude 会一次性问你以下 3 个问题：

1. **题目文件名** → 告诉它文件名（支持 PDF/Word/图片等格式）。主代理没有多模态能力，图片文件或 PDF/Word 中的示意图/图表会自动派子代理（model: "opus"）识别
2. **格式要求文件名** → 如 `format2025.pdf`
3. **是否有附件数据** → 有就告诉它 Excel 文件名

Claude 会用 PDF 工具读取题目，用 Excel 工具读取数据，然后向你确认：
- 题目有几个小问
- 每个小问的核心要求
- 附件数据的结构和规模

**确认无误后回复"确认"，进入下一步。**

### 步骤 3：探索建模思路（/mmstep2）

```
/mmstep2
```

Claude 会使用 brainstorming 技能，**一个问题一个问题地**分析：

- 理解问题本质
- 识别约束条件
- 探索 2-3 种建模思路
- 说明每种思路的优劣
- 给出推荐方案

**每分析完一个问题，你需要确认方向，Claude 才继续下一个。**

全部分析完成后，Claude 会产出设计文档（`docs/superpowers/specs/`）。

### 步骤 4：制定计划（/mmstep3）

```
/mmstep3
```

Claude 会使用 writing-plans 技能，将整个项目拆分为可执行的子任务：

- 每个子任务 2-10 分钟可完成
- 按问题分文件规划代码结构
- 设计论文大纲
- 标注可并行的任务和需要 TDD 的核心算法

产出计划文档（`docs/superpowers/plans/`），**确认后再进入建模阶段。**

### 步骤 5：建立数学模型（/mmstep4）

```
/mmstep4
```

Claude 会先读取前序产出（设计文档 + 计划文档），主代理没有多模态能力，遇到图像内容会自动派子代理（model: "opus"）处理，然后按顺序执行：

0. **附件检查**：检查数据文件夹，读取附件数据，确认数据充足性
1. **验证选型**：获取 mmstep2 已选模型（只验证不推翻，发现问题报告用户决定），用建模设计理论验证（三原则、复杂度控制），参考 assets/ 算法资源库，组织成前置合同格式（子问题映射、风险、交付规格）
2. **搜索文献**：为已选模型找学术论文支撑
3. **执行建模分析**：按工作流程推导公式、设计算法、建立术语表，推荐模型说明图方案（2-3 种），确认后按 nature-figure 的设计规则绘图并做 QA 检查
4. **缺失输入处理**：检查并处理数据不足、模型不适用等问题
5. **质检交付**：按质检清单逐项检查

产出文件：
- `题目分析报告.md` — 完整的建模分析（项目根目录）
- `术语表格.md` — 中英文术语对照（项目根目录）
- 模型说明图（`figures/` 目录）

**每个步骤完成后 Claude 会暂停，确认后再进入下一步。**

### 步骤 6：编写求解代码（/mmstep5）

```
/mmstep5
```

Claude 会先读取建模阶段产出（题目分析报告、术语表）和计划文档（代码结构规划、TDD 标记），确认执行策略（TDD 范围、是否并行、模型检查规则），主代理没有多模态能力，遇到图像内容会自动派子代理（model: "opus"）处理，然后按顺序执行：

0. **环境检查**：确认 Python 版本和依赖库
1. **编写求解代码**：按工作流程编写代码、运行、输出结果表格（参考 assets/ 算法资源库）
2. **结果验证**：数值合理性检查、边界条件、与预期对比
3. **三类可视化规划**：每个问题需要原始数据图(EDA)、中间产物图(模型过程)、最终结果图(核心结论)，每类至少 1-2 张
4. **参考图表模板**：选择对应的代码模板
5. **可视化规范 + 绘图**：通过 nature-figure skill 全流程创建图表（五点合同 + 设计理论 + QA 检查）
6. **HTML 图表面板**：基于模板生成导航面板
7. **生成 README**：项目简介、运行说明
8. **质检交付**：逐项检查后交付

产出文件：
- `code/` — 各问题的求解代码（如 `code/问题1_求解.py`）
- `results/结果表格.csv` — 计算结果
- `figures/` — 可视化图表（SVG + PNG）+ HTML 面板

**执行策略（开始前确认）：** 如果配置了 TDD，Claude 会对核心算法函数先写测试再实现。如果多个问题相互独立，Claude 会问你是否并行求解。

**模型检查（执行中持续）：** 在实现公式时（维度不一致、除零风险）、运行代码后（NaN/Inf/不合理值）、优化求解时（不收敛、约束冲突）、对比验证时（结果偏差过大）自动检查。小问题直接修正，大问题暂停等你确认。

### 步骤 7：撰写论文（/mmstep6）

```
/mmstep6
```

Claude 会先确认执行策略（是否并行撰写各章节），主代理没有多模态能力，遇到图像内容会自动派子代理（model: "opus"）处理，然后按顺序执行：

1. **构建论证框架**：一句话论证模板 + Claim-Evidence 映射表，缺失项用占位符标注
2. **读取章节模板**，规划论文结构（摘要六要素、引言五要素、结果证据阶梯）
3. **读取写作规范**（禁止第一人称、去 AI 味 8 种模式）
4. **读取进阶技巧 + 自审框架**（段落流检查、动词校准、四轮自审）
5. 问你论文格式（Word / LaTeX），选择对应模板
6. 读取前置产出（分析报告、术语表、结果表格、图表），如有缺失会及时反馈
7. 撰写论文初稿（正文约 18 页 + 附录完整，总页数 25-30 页以上），每张图表配 100+ 字分析
8. 美赛时读取英文化工作流，将中文论文转化为规范英文
9. 执行外部审查：
    - **nature-reviewer**：按五维度评估（原创性、科学重要性、跨学科读者兴趣、技术可靠性、可读性），输出 3 份审稿报告 + 跨审稿综合，保存到 `paper/审稿报告.md`
    - **nature-figure**：图表 SCI/Nature 标准检查
10. **Claim-Evidence 映射终检**：确认每个论点都有证据支撑
11. **stop-slop**：去 AI 味 + 润色
12. **格式检查与最终输出**，产出论文到 `paper/` 目录

产出文件：
- `paper/论文.docx`（Word 版）或 `paper/论文.tex` + `paper/论文.pdf`（LaTeX 版）
- `paper/审稿报告.md`（3 份审稿报告 + 跨审稿综合 + 风险声明）

**每个步骤完成后 Claude 会暂停，确认后再进入下一步。**

### 步骤 8：打包支撑材料（/mmstep7）

```
/mmstep7
```

Claude 会按顺序执行：

1. 收集支撑材料文件：源程序、外部查阅数据、中间结果图表、分析文档
2. 检查合规性：确认无参赛者身份、学校、赛区信息，无承诺书和编号页
3. 检查完整性：核对论文附录引用是否全部包含，源程序是否可运行
4. 将支撑材料文件列表写入论文附录（如无支撑材料则注明"本论文没有支撑材料"）
5. 压缩为 ZIP 或 RAR（不超过 20MB）

产出：
- `支撑材料.zip`（项目根目录，不超过 20MB）
- 论文附录已更新文件列表

**每个步骤完成后 Claude 会暂停，确认后再进入下一步。**

**注意：支撑材料中不能有显示参赛者身份和所在学校及赛区的信息。**

## 四、关键技巧

### 4.1 如何给 Claude 有效的指令

**好的指令：**
```
/mmstep5
使用 Python，对附件1和附件2计算碳化硅外延层厚度，
用峰值间距法，结果写入 结果表格.csv
```

**不好的指令：**
```
帮我算一下
```

**原则：说清楚做什么、用什么方法、输出到哪里。**

### 4.2 如何处理 Claude 卡住

如果 Claude 长时间无响应：

1. 按 `Ctrl+C` 中断
2. 说"继续"或"你卡了，继续"
3. 如果反复卡住，说"换个方法试试"

### 4.3 如何让 Claude 改进结果

```
结果不对，请检查模型公式是否正确
```

```
R² 太低了，换个拟合方法试试
```

```
论文太短了，扩充到 18 页正文
```

### 4.4 如何查看中间产物

Claude 的每一步都会产出文件，你可以随时查看：

- **结果表格**：用 Excel 或文本编辑器打开 `results/结果表格.csv`
- **图表**：直接用文件管理器打开 `figures/` 下的 PNG/SVG 文件
- **HTML 面板**：用浏览器打开 `figures/图表面板.html`

### 4.5 并行执行

如果问题之间独立（没有数据依赖），Claude 会问你是否并行求解：

```
问题2和问题3相互独立，是否使用并行求解？
```

同意后，Claude 会为每个问题派发独立的子 agent 并行工作，完成后自动审查。

### 4.6 中途修改方案

任何时候都可以说：

```
换个方法，用XXX模型替代
```

Claude 会重新调整代码和论文。

## 六、常见问题

### Q: Claude Code 和网页版 Claude 有什么区别？

Claude Code 是命令行工具，可以直接读写你电脑上的文件、运行代码、编译 LaTeX。网页版只能对话，不能操作文件。

### Q: 需要什么版本的 Claude？

Claude Code 默认使用最新版 Claude 模型。可通过 `/model` 命令切换。

### Q: 代码报错了怎么办？

直接把报错信息粘贴给 Claude，它会自动修复。不需要自己调试。

### Q: 论文格式不对怎么办？

告诉 Claude 具体哪里不对，比如：
```
字号应该是小四，行距1.5倍
```

### Q: 可以中途修改方案吗？

可以。任何时候都可以说：
```
换个方法，用XXX模型替代
```

Claude 会重新调整代码和论文。

### Q: 附录代码太多/太少？

告诉 Claude：
```
附录把所有代码都放进去，越完整越好
```

或：
```
附录只放核心算法代码
```

### Q: 如何保存进度？

Claude Code 的所有产出都是文件，自动保存在工作目录中。即使退出重开，文件还在。

### Q: 可以同时开多个 Claude Code 吗？

可以。每个终端窗口是一个独立的 Claude Code 会话。

## 七、实战小贴士

1. **先读题再动手**：让 Claude 先完整读取题目和数据，确认理解后再开始建模。

2. **逐问确认**：每个问题分析完后确认方向，避免后面返工。

3. **及时纠偏**：发现结果不对立刻告诉 Claude，不要等全部做完。

4. **检查数值**：论文中的数值要和代码输出一致，可以让 Claude 做三方比对。

5. **图表很重要**：国赛评阅时图表是第一印象，确保清晰、规范、有分析文字。

6. **附录要完整**：代码必须可运行，支撑材料必须齐全。

7. **提前练习**：比赛前用一道真题走一遍完整流程，熟悉各命令的效果。
